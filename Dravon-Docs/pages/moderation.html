Moderation Commands

Powerful moderation tools to manage your server effectively with advanced features

Setup Guide

1

Basic Setup

No initial setup required! Moderation commands work out of the box with proper permissions.

# Ensure Dravon has these permissions:
• Manage Messages
• Kick Members  
• Ban Members
• Manage Roles (for mute)
• View Audit Log
2

Configure Mute Role

Create a muted role or let Dravon create one automatically when first using mute commands.

>mute @user 10m Spamming
# Dravon will automatically create a "Muted" role if none exists
# The role will be configured to deny Send Messages in all channels
3

Set Up Logging

Configure moderation logs to track all moderation actions and maintain server security.

>logs setup
# Select moderation logs channel
# All bans, kicks, mutes, and warnings will be logged

Commands List

Ban Commands

>ban <user> [reason]

Permanently ban a user from the server

>unban <user_id>

Remove a ban from a user using their ID

>tempban <user> <time> [reason]

Temporarily ban a user (auto-unbans after time)

>massban <user1> <user2> [reason]

Ban multiple users at once

Kick Commands

>kick <user> [reason]

Remove a user from the server

>masskick <user1> <user2> [reason]

Kick multiple users at once

Mute Commands

>mute <user> [time] [reason]

Mute a user for a specified time

>unmute <user>

Remove mute from a user

>tempmute <user> <time> [reason]

Temporarily mute with auto-unmute

Warning System

>warn <user> [reason]

Issue a warning to a user

>warnings <user>

View all warnings for a user

>clearwarns <user>

Clear all warnings for a user

>delwarn <user> <warning_id>

Delete a specific warning

Cleanup Commands

>purge <amount>

Delete multiple messages at once (max 100)

>purge <amount> <user>

Delete messages from a specific user

>purge <amount> bots

Delete messages from bots only

Timeout Commands

>timeout <user> <time> [reason]

Discord native timeout (max 28 days)

>untimeout <user>

Remove timeout from a user

Usage Examples

Basic Moderation

# Ban a user with reason
>ban @spammer Repeated spam after multiple warnings

# Kick a user  
>kick @troublemaker Inappropriate behavior in chat

# Mute for 30 minutes
>mute @user 30m Please follow server rules

Time-based Actions

# Different time formats
>mute @user 10s    # 10 seconds
>mute @user 5m     # 5 minutes  
>mute @user 2h     # 2 hours
>mute @user 1d     # 1 day
>tempban @user 7d  # 7 day temporary ban

Bulk Actions

# Delete last 50 messages
>purge 50

# Delete last 20 messages from specific user
>purge 20 @user

# Delete bot messages only
>purge 30 bots

# Mass ban multiple users
>massban @user1 @user2 @user3 Raid attempt

Pro Tips

Always Use Reasons

Provide clear reasons for all moderation actions. They appear in audit logs, DMs to users, and help track patterns of behavior.

Temporary Actions

Use temporary mutes and bans when appropriate. They automatically expire without manual intervention, reducing moderator workload.

Role Hierarchy

Ensure Dravon's role is above the roles you want to moderate. Check role positions in Server Settings > Roles.

Team Coordination

Use the warning system to communicate with other moderators about problematic users and track escalation.

Monitor Patterns

Review moderation logs regularly to identify trends and adjust your server rules or auto-mod settings accordingly.

Consistent Enforcement

Apply rules consistently across all members. Document your moderation guidelines for your team to follow.
