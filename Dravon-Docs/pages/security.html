Security & AntiNuke v6.0

Advanced server protection system with intelligent threat detection and prevention

Setup Guide

1

Enable AntiNuke

Activate the AntiNuke system to protect your server from malicious activities.

>antinuke enable
# Enables basic protection with default settings
# Protects against mass bans, kicks, channel deletion, etc.
2

Configure Whitelist

Add trusted users who can bypass AntiNuke protection.

>whitelist add @user
# Adds user to whitelist (bypasses all protection)

>extraowner add @user  
# Adds user as extra owner (highest permissions)
3

Set Protection Limits

Configure action limits and punishment settings for optimal protection.

>antinuke config
# Opens interactive configuration menu
# Set limits for bans, kicks, channel actions, etc.
# Configure punishment: ban, kick, or strip roles

Commands List

AntiNuke Control

>antinuke enable

Enable AntiNuke protection system

>antinuke disable

Disable AntiNuke protection system

>antinuke config

Open interactive configuration menu

>antinuke status

View current AntiNuke settings and status

Whitelist Management

>whitelist add <user>

Add user to AntiNuke whitelist

>whitelist remove <user>

Remove user from whitelist

>whitelist list

View all whitelisted users

>whitelist clear

Remove all users from whitelist

Extra Owner System

>extraowner add <user>

Add user as extra owner (highest permissions)

>extraowner remove <user>

Remove extra owner status

>extraowner list

View all extra owners

Protection Settings

>antinuke punishment <type>

Set punishment: ban, kick, or strip

>antinuke limit <action> <number>

Set action limits (bans, kicks, etc.)

>antinuke logs <channel>

Set AntiNuke logging channel

Security Logs

>antinuke recent

View recent AntiNuke actions

>antinuke stats

View protection statistics

>antinuke reset

Reset all AntiNuke settings

Manual Actions

>nuke <user>

Manually trigger AntiNuke punishment

>unnuke <user>

Reverse AntiNuke punishment

Configuration Examples

Basic Setup

# Enable AntiNuke with default settings
>antinuke enable

# Add trusted moderators to whitelist
>whitelist add @moderator1
>whitelist add @moderator2

# Set logging channel
>antinuke logs #security-logs

Advanced Configuration

# Set custom limits
>antinuke limit ban 3      # Max 3 bans per minute
>antinuke limit kick 5     # Max 5 kicks per minute
>antinuke limit channel 2  # Max 2 channel deletions

# Set punishment to ban
>antinuke punishment ban

Emergency Response

# Manual nuke a compromised account
>nuke @compromised_user

# Check recent AntiNuke actions
>antinuke recent

# View protection statistics
>antinuke stats

Security Best Practices

Whitelist Carefully

Only whitelist users you completely trust. Whitelisted users can bypass all AntiNuke protection.

Configure Limits

Set appropriate action limits based on your server size and activity. Too strict may cause false positives.

Monitor Logs

Regularly check AntiNuke logs to understand threats and adjust settings accordingly.

Role Hierarchy

Ensure Dravon's role is at the top of your server's role hierarchy for maximum protection.

Backup Settings

Document your AntiNuke configuration. Use >antinuke status to export current settings.

Regular Updates

Keep your whitelist updated. Remove users who no longer need elevated permissions.
